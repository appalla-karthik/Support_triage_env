# Support Triage Env

`Support_triage_env` is an OpenEnv-compatible customer support triage simulator.
It is designed to evaluate and train agents on a realistic business workflow:

- reading a queue of support tickets
- classifying the issue correctly
- prioritizing urgent work
- drafting policy-safe customer replies
- routing tickets to the right internal team
- deciding whether to escalate or resolve

This documentation site lives at the workspace root and focuses on the project in
`Support_triage_env/`.

## What already exists in the repo

The current project is more than a sketch:

- a typed action, observation, reward, and state model
- a simulator with `reset`, `step`, and `state`
- ten seeded task families with deterministic graders
- an OpenEnv/FastAPI server
- a competition-style `inference.py` entrypoint
- synthetic dataset generation, trajectory generation, and a lightweight classifier baseline
- an Unsloth + TRL Colab notebook for minimal post-training
- tests for simulator, inference policy, logging, and data generation

## Recommended submission framing

The best way to pitch this project is:

> A policy-aware support operations environment where an agent must manage a
> partially observable enterprise queue, coordinate correct routing and customer
> communication, and optimize for safe outcomes under delayed business impact.

That framing is grounded in what the repo already does, while leaving room for a
stronger pivot into longer-horizon and multi-actor workflows.
